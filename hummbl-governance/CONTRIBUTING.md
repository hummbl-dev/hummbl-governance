## Branching
- Work in feature branches prefixed with `feature/` or `fix/`.

## Pull Requests
- All changes must go through PRs targeting `main`.
- Include MRCC hashes for new artifacts.
- Ensure compliance with existing templates.

## Review
- Require one approving review before merge.
- Verify artifact paths and hashes are correct.
