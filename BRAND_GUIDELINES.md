# Brand Guidelines

Status: public boundary
Last updated: 2026-07-03

This repository is a code and governance artifact. The Apache 2.0 license
applies to code reuse. HUMMBL brand use is controlled separately.

## Naming

Use `hummbl-governance` when referring to the Python package or repository.
Use `HUMMBL` when referring to the organization.

Do not rename forks, integrations, hosted services, or derivative packages in a
way that makes them look official without written permission.

## Public Wording

Public descriptions should be evidence-backed and specific:

- Prefer "maps controls to governance events" over "certifies compliance".
- Prefer "CI-tested on Python 3.11 through 3.13" over broad runtime claims.
- Prefer "engineering mapping" over "coverage" when no third-party attestation
  exists.
- Prefer "planned", "draft", or "source-candidate" when evidence is not final.

## Visual Identity

Do not use HUMMBL logos, product marks, screenshots, or visual assets in a fork,
commercial service, template, marketplace listing, or publication without
permission.

## Attribution

Do not add vendor attribution strings to commits, PRs, issues, docs, or release
artifacts. The operator's work is the operator's IP.
