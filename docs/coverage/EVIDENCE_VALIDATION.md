## How to harden a row

A Fulfilled row currently failing validation can be hardened by ONE of:

1. **Add an Evidence reference** — backtick-quote a file path (e.g., `` `hummbl_governance/kill_switch.py` ``) or command (e.g., `` `compliance_mapper --framework eu-ai-act --export annex-iv` ``) in the row's Evidence column.
2. **Update legacy reference** — if the row cites a founder-mode-layout path like `services/kill_switch_core.py`, either (a) update it to the canonical hummbl-governance path (`hummbl_governance/kill_switch.py`), or (b) add the legacy mapping to `_MODULE_ALIASES` in `compliance_mapper.py`.
3. **Downgrade the row** — if the primitive does not yet exist as shipped code, change the state glyph from Fulfilled to Partial with the missing-half named.
4. **Mark as Tier-2/External** — if the evidence is an admitted dependency (e.g., `services/c2pa_mcp`) or lives in another repo (e.g., `_state/coordination/messages.tsv` from founder-mode), use the existing `_MODULE_ALIASES` markers `TIER2_ADMITTED:` / `EXTERNAL:`.

After hardening, re-run validation:

```bash
python -m hummbl_governance.compliance_mapper --validate docs/coverage/<matrix>.md
python scripts/build_evidence_validation_report.py
```
