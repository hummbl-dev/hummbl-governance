# Source-Candidate Policy

Status: public boundary
Last updated: 2026-07-03

Source-candidate material is useful research that has not yet been promoted to
verified public claim status.

## Definition

A source-candidate may be:

- benchmark research;
- draft prior art;
- an unverified external pattern;
- an internal observation awaiting receipts;
- a planned implementation lane;
- a claim copied from older docs that needs current verification.

## Required Labels

Source-candidate material must be clearly marked as one of:

- `source-candidate`
- `draft`
- `planned`
- `unverified`
- `needs receipt`

- `needs-boundary-review`

## Promotion To Verified Claim

A source-candidate can become a verified public claim only when it has:

- current primary-source evidence;
- date verified;
- command, URL, commit, CI run, release, or artifact receipt;
- public/private boundary review;
- license/trademark/brand review when relevant;
- operator approval for public promotion when the claim is consequential.

## Non-Use

Do not use source-candidate material as:

- marketing copy;
- benchmark proof;
- release proof;
- security assurance;
- compliance assurance;
- customer-facing commitment.

Until promoted, it remains planning context only.
