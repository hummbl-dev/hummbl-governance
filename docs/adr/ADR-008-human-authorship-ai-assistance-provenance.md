# ADR-008 — Human authorship and AI assistance provenance

- **Status:** proposed
- **Date:** 2026-06-30
- **Decision owner:** Reuben Bowlby
- **Steward:** HUMMBL Research Institute
- **Supersedes:** none
- **Superseded by:** none

## Context

HUMMBL repositories are increasingly operated with AI agents that research, draft, review, and prepare patches. That assistance is operationally useful, but durable Git authorship metadata can create avoidable ambiguity around IP ownership, patent inventorship, copyright authorship, trade-secret handling, and future diligence.

Current U.S. patent and copyright posture centers natural-person inventorship and human authorship. Vendor terms may address output ownership as between vendor and customer, but they do not eliminate the need for a clean human project record.

## Decision

The HUMMBL Repo Standard now separates AI assistance receipts from Git authorship provenance.

AI agents may assist with research, drafting, review, patch preparation, and coordination. They must not be credited as commit authors or co-authors. Commit author and committer identities must be human/operator identities or approved human-controlled service identities.

Commit messages must not contain AI authorship trailers or equivalent credit lines, including:

- `Co-authored-by` entries naming AI agents, model providers, agent products, or bots
- `Generated-by`
- `Authored-with`
- `Authored-by`
- `Created-with`
- prose credit lines such as `Implemented by Devin` or `Generated by Claude`

Agent activity may be recorded in internal bus messages, ledgers, handoffs, receipts, PR notes, or review notes. Those records support operational traceability without contaminating Git authorship metadata.

## Enforcement

This repo adds `tools/provenance_lint.py`, a stdlib-only linter that rejects AI authorship fingerprints in commit messages. The local `.githooks/commit-msg` hook invokes it before Conventional Commit validation.

Fleet rollout should add the same rule to repo-local `AGENTS.md` files and use equivalent commit-message or CI checks in active repositories.

## Consequences

- Git history remains a human-authored engineering and IP record.
- AI assistance remains traceable through internal operational records.
- IP-sensitive repositories can adopt stricter controls without weakening the global baseline.
- Existing historical commits are not rewritten by this ADR; remediation is a separate fleet audit decision.
