AG01: Excessive Agency

Risk: Agents perform actions beyond their intended scope, accessing unauthorized tools, data, or system capabilities without proper constraints.

Coverage: FULL

ModuleMechanism
delegation_token.pyScope-constrained capability tokens with explicit permission grants. Agents cannot act outside delegated scope.
delegation_context.pyChain depth enforcement (≤3) prevents unbounded delegation cascades. 7-state lifecycle tracks delegation from CREATED through COMPLETED or REVOKED.
kill_switch.py4-mode graduated shutdown allows immediate restriction of agent capabilities at any severity level.

Gaps: None. The delegation token + context + kill switch combination provides defense-in-depth against excessive agency. Scope is cryptographically enforced, delegation depth is bounded, and emergency shutdown is always available.

AG02: Inadequate Sandboxing

Risk: Agents execute in environments without proper isolation, allowing them to affect other systems, access restricted resources, or escape containment.

Coverage: PARTIAL

ModuleMechanism
circuit_breaker.pyFailure isolation on external boundaries prevents cascade failures across service boundaries.
kill_switch.pyHALT_ALL and EMERGENCY modes provide containment when sandbox violations are detected.

Gaps: The framework provides logical isolation (circuit breakers, kill switch) but not process-level or OS-level sandboxing. Container isolation, filesystem restrictions, and network segmentation are outside the scope of a Python governance library and must be provided by the deployment environment.

AG03: Broken Authentication and Authorization

Risk: Weak or missing authentication allows agents to impersonate other agents, escalate privileges, or bypass access controls.

Coverage: FULL

ModuleMechanism
delegation_token.pyHMAC-SHA256 signed tokens provide cryptographic authentication. Tokens cannot be forged without the signing key. Each token carries issuer, subject, scope, and expiry.
delegation_context.pyChain-of-custody tracking ensures every delegation is traceable to the original authority. State machine prevents re-use of consumed tokens.
governance_bus.pyAll authentication events are logged to the append-only audit trail for post-hoc verification.

Gaps: None. Token-based authentication with HMAC-SHA256 signatures, combined with append-only audit logging, covers both real-time enforcement and forensic review.

AG04: Insufficient Logging and Monitoring

Risk: Inadequate logging prevents detection of agent misbehavior, unauthorized actions, or security incidents.

Coverage: FULL

ModuleMechanism
governance_bus.pyAppend-only JSONL log captures every governance event with timestamp, agent identity, event type, and payload. Events cannot be deleted or modified.
circuit_breaker.pyState transitions (CLOSED → OPEN → HALF_OPEN) are logged with failure counts and recovery timestamps.
kill_switch.pyMode changes are logged with justification and triggering agent.

Gaps: None. The governance bus provides comprehensive, immutable audit logging. Combined with circuit breaker and kill switch state change logging, the framework provides full observability of governance events.

AG05: Unsafe Output Handling

Risk: Agent outputs are consumed by downstream systems without validation, enabling injection attacks, data corruption, or unintended actions.

Coverage: PARTIAL

ModuleMechanism
models.pyTyped data models enforce output schema. AgentHealthReport, CostProjection, and BriefingContent provide structured output contracts.
governance_bus.pyOutput events logged for post-hoc review and anomaly detection.

Gaps: The framework provides schema-level output validation through data models but does not include content-level output sanitization (e.g., stripping prompt injection payloads, validating generated code safety). Content filtering and output sanitization must be implemented at the application layer.

AG06: Prompt Injection

Risk: Malicious inputs manipulate agent behavior through prompt injection, causing agents to ignore instructions, leak data, or perform unauthorized actions.

Coverage: NONE

ModuleMechanism
No modules directly address prompt injection.

Gaps: Prompt injection defense is outside the scope of a governance framework. It requires input filtering, prompt hardening, and model-level mitigations that operate at the LLM inference layer, not the governance layer. The framework's delegation tokens prevent unauthorized actions even if prompt injection succeeds, providing defense-in-depth. However, direct prompt injection detection and prevention must be implemented separately.

AG07: Lack of Transparency

Risk: Agent decision-making processes are opaque, making it impossible to audit, explain, or reproduce agent behavior.

Coverage: FULL

ModuleMechanism
governance_bus.pyComplete event history provides full transparency into agent governance decisions. Every delegation, state change, and policy enforcement is recorded.
delegation_context.py7-state lifecycle makes delegation decisions explicit and traceable. Each state transition is a visible, auditable event.
compliance_mapper.pyCross-framework mapping provides transparency into which controls are satisfied and which have gaps.

Gaps: None. The append-only governance bus combined with explicit delegation state machines provides full transparency into governance layer decisions. Note: transparency into model reasoning (why the LLM chose a particular action) is outside scope.

AG08: Improper Error Handling

Risk: Agents handle errors poorly, exposing sensitive information, entering undefined states, or failing to recover gracefully.

Coverage: PARTIAL

ModuleMechanism
circuit_breaker.pyAutomatic failure detection and recovery. After threshold failures, the breaker opens and prevents cascading errors. Half-open state tests recovery before full resumption.
kill_switch.pyGraduated error response: HALT_NONCRITICAL for non-critical failures, HALT_ALL for systemic issues, EMERGENCY for catastrophic failures.

Gaps: The framework handles errors at the governance boundary (circuit breakers, kill switch) but does not prescribe error handling within agent logic itself. Application-level error handling patterns (retry policies, fallback strategies, error classification) must be implemented by the consuming application.

AG09: Insecure Third-Party Integration

Risk: Agents integrate with third-party services without proper security controls, creating supply chain risks and data exposure.

Coverage: PARTIAL

ModuleMechanism
circuit_breaker.pyEvery external integration wrapped in a circuit breaker. Failure in one third-party service does not cascade to others.
delegation_token.pyScope constraints limit which external services an agent can access through delegated authority.
Framework designStdlib-only architecture eliminates third-party supply chain risk for the governance framework itself.

Gaps: The framework provides failure isolation (circuit breakers) and access control (delegation tokens) for third-party integrations but does not include API key management, secret rotation, or third-party service authentication. These concerns must be handled by the deployment infrastructure.

AG10: Uncontrolled Scaling

Risk: Agents spawn unbounded child processes, consume excessive resources, or create runaway loops without rate limiting.

Coverage: NONE

ModuleMechanism
delegation_context.pyChain depth ≤3 limits delegation depth but not breadth (fan-out).

Gaps: The framework limits delegation chain depth but does not limit breadth (number of concurrent delegations) or resource consumption (CPU, memory, API calls). Rate limiting, resource quotas, and concurrency controls must be implemented at the orchestration layer. The kill switch provides emergency containment but not proactive scaling controls.

Recommendations for Full Coverage

  1. AG06 — Prompt Injection: Implement an input sanitization layer upstream of the governance framework. Consider a pluggable filter interface that can be composed with delegation token validation. Integration with model-level prompt hardening (system prompt anchoring, instruction hierarchy) provides defense-in-depth.
  2. AG10 — Uncontrolled Scaling: Add a ConcurrencyGovernor module with configurable fan-out limits, per-agent rate limiting, and resource budget enforcement. The delegation context already tracks chain depth; extending it to track breadth (concurrent active delegations per issuer) is a natural evolution.
  3. AG02 — Sandboxing (upgrade to FULL): Provide deployment reference architectures (container profiles, seccomp policies, network policies) that complement the logical isolation provided by circuit breakers. Document recommended OS-level isolation patterns for each deployment target.
  4. AG05 — Output Handling (upgrade to FULL): Add an OutputValidator module with pluggable content filters. Schema validation via models.py handles structure; content validation (injection detection, PII scanning, code safety analysis) requires a dedicated component.
